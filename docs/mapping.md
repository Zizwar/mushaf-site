## 1. المقدمة والمشكلة

### لماذا هذا الموضوع نادر على الإنترنت؟

اختلاف عدد الآيات بين رواية ورش ورواية حفص موثّق في كتب علم القراءات، لكن لا يوجد — في حدود ما بحثنا — مصدر رقمي يوثّق **نقاط الانتقال بدقة آية بآية** بصيغة قابلة للبرمجة. معظم المصادر تذكر فقط أن "العدّ الكوفي يختلف عن العدّ المدني" دون تحديد السورة والآية بالضبط.

### السياق التقني

التطبيق يعرض مصحف ورش (رواية ورش عن نافع) ويدعم تشغيل الصوت وعرض التفسير. مصدر التفسير هو API من جامعة الملك سعود (KSU) الذي يستخدم ترقيم حفص (العدّ الكوفي). عند الضغط على آية في وضع ورش، كانت الدالة ترسل رقم الآية كما هو إلى API، مما يعطي تفسير آية مختلفة تماماً في السور التي يختلف فيها الترقيم.

### أرقام المشكلة

| الرواية | العدّ | المصدر في التطبيق |
|---------|-------|------------------|
| ورش (نافع — مدني أخير) | 6214 آية | SQLite DB: `aya_audio` |
| حفص (عاصم — كوفي) | 6236 آية | `src/data/ayatJson.js` |
| الفرق | 22 آية | موزّعة على 50 سورة |

## 2. المفاهيم الأساسية

### 2.1 أنواع الاختلاف بين العدّين

**الدمج (Merge) — ورش يجمع ما فرّقه حفص:**
حفص يعدّ آيتين منفصلتين، ورش يعدّهما آية واحدة.
```
H[n] + H[n+1]  →  W[k]        (ورش له آية أقل)
```

**التقسيم (Split) — ورش يفرّق ما جمعه حفص:**
حفص يعدّ آية واحدة طويلة، ورش يعدّها آيتين.
```
H[n]  →  W[k] + W[k+1]        (ورش له آية أكثر)
```

**الفواتح (Fawatih Merge):**
الحروف المقطعة كـ"الم" و"حم" تُعدّ في ورش آية مع ما بعدها، بينما حفص يعدّها مستقلة. هذا النوع الأكثر تكراراً ويظهر في 16 سورة.

### 2.2 مفهوم الإزاحة (Offset)

الإزاحة `offset` هي الفرق بين رقم الآية في ورش ورقمها المقابل في حفص:
```
W[n]  =  H[n + offset]
```
- `offset = 0`: تطابق مباشر
- `offset = +1`: ورش دمج سابقاً (حفص أمامه بآية)
- `offset = -1`: ورش قسّم سابقاً (حفص خلفه بآية)

الإزاحة تتغير عند كل نقطة دمج أو تقسيم وتبقى ثابتة حتى النقطة التالية.

## 3. الحل التقني

### 3.1 هيكل الكود

```typescript
// src/utils/tafsir.ts

const WARSH_EXCEPTIONS: Record<number, ExceptionType> = { ... }

export function warshToHafsAyahs(sura: number, warshAya: number): number[]
```

الدالة `warshToHafsAyahs` تُرجع **مصفوفة** (وليس رقماً واحداً) لأن آية ورش الواحدة قد تقابل آيتين في حفص (عند الدمج).

### 3.2 أنواع الاستثناءات

```typescript
type ExceptionType =
  | { type: "manual"; map: Record<number, number[]> }
  | { type: "offset"; mergeAtWarshAyah: number; hafsTargetsToMerge: number[]; offsetForSubsequent: number }
  | { type: "advanced"; explicit_map: Record<number, number[]>; ranges: Range[] }
```

| النوع | الاستخدام |
|-------|----------|
| `manual` | الفاتحة فقط — خريطة يدوية كاملة |
| `offset` | دمج فاتحة بسيط عند W1، إزاحة +1 لكل ما بعدها |
| `advanced` | أي سورة بها انتقالات متعددة أو داخلية |

### 3.3 مثال على `advanced`

```typescript
// مريم (19): W=99, H=98
// دمج عند البداية + تقسيمان داخليان
19: {
  type: "advanced",
  explicit_map: {
    1: [1, 2],      // W1 = H1+H2  (دمج الفاتحة)
    40: [41],       // W40 = H41   (أول نصف التقسيم)
    41: [41],       // W41 = H41   (ثاني نصف التقسيم)
    75: [75],       // W75 = H75   (أول نصف التقسيم الثاني)
    76: [75],       // W76 = H75   (ثاني نصف التقسيم الثاني)
  },
  ranges: [
    { warshStart: 2,  warshEnd: 39, hafsOffset:  1 },
    { warshStart: 42, warshEnd: 74, hafsOffset:  0 },
    { warshStart: 77, warshEnd: 99, hafsOffset: -1 },
  ],
}
```

## 5. منهجية التحقق

### 5.1 التحقق من آخر آية (last-aya validation)

أكثر الاختبارات موثوقية: التحقق من أن `warshToHafsAyahs(W_last) == H_last` لكل سورة:

```python
mapped = warsh_to_hafs(exc, w_last)
assert mapped[-1] == h_last  # يجب أن تتطابق
```

نجح لجميع 56 سورة في `WARSH_EXCEPTIONS`.

### 5.2 التحقق من المحتوى (content validation)

مقارنة نص الآيات الفعلي عند نقاط الانتقال باستخدام تطبيع عربي:

```python
def normalize(text):
    # إزالة التشكيل، توحيد أشكال الألف، الياء، الهاء...
    text = re.sub(r'[أإآٱ]', 'ا', text)
    text = text.replace('ى','ي').replace('ے','ي')
    text = re.sub(r'[^\u0621-\u064A\s]', '', text)  # حروف عربية فقط
```

**تنبيه مهم:** مقارنة المحتوى بين ورش وحفص تنتج **false positives** كثيرة بسبب اختلاف الرسم العثماني بين الروايتين. مثلاً: ورش يكتب "السموت" وحفص يكتب "السماوات"، وهو اختلاف مقصود في الرسم وليس خطأ. لذلك استُخدم التحقق من المحتوى فقط **لتأكيد نقاط الانتقال** وليس للمقارنة العامة.

## 8. ملاحظات للمطورين

### اختلاف الرسم ليس خطأ

عند مقارنة نص ورش بنص حفص برمجياً، الاختلافات الإملائية التالية **طبيعية ومتوقعة**:

| ورش (مدني) | حفص (كوفي) | ملاحظة |
|-----------|-----------|--------|
| السموت | السماوات | رسم عثماني مختلف |
| ءامنوا | آمنوا / امنوا | اختلاف كتابة الهمزة |
| اهلكنها | اهلكناها | اختلاف في ضمير الجمع |
| يراءون | يرائون | اختلاف رسم الهمزة |
| ے (يه باري) | ي | حرف مختلف في ورش |
| ۞ (ربع الحزب) | — | علامة موجودة في DB ورش |

لذا أي خوارزمية تقارن نصوص الروايتين تحتاج تطبيعاً مخصصاً يراعي هذه الفوارق.

### منطق الصوتيات

الدالة `warshToHafsAyahs()` تُستخدم في **وضعين**:

1. **التفسير**: دائماً — KSU API يستخدم ترقيم حفص
2. **الصوت (CDN)**: فقط عند `moqriId.startsWith("warsh_")` — لأن ملفات الصوت على السيرفر مرقّمة بترقيم حفص حتى للقراء الذين يقرؤون بورش
3. **الصوت (DB)**: لا يحتاجها — الكوشي والقزابري مرقّمان بترقيم ورش مباشرة
